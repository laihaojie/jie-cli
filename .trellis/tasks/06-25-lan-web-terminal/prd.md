# 局域网网页终端远程命令执行

## Goal

本机运行 jie CLI 后，局域网内任意设备打开浏览器即可获得一个真实交互式终端（等同在本机开 shell），用于远程执行命令。替换当前「POST 单条命令 → 返回 stdout」且仅监听 127.0.0.1 的 bridge 实现。

## Background

- 现状：`src/bridge.ts` 是单次命令执行模型（`handlePost` 执行 cmd 返回 stdout），监听 127.0.0.1:32677，无 WebSocket，无法被局域网其他设备访问，也不是真终端。
- `startServer()`（`src/commands/server.ts`）在每次 CLI 调用时于后台 detached 拉起 bridge；当前**没有** `jie server` 子命令（执行任意 jie 命令即自动拉起，即用户印象中的「jie server」）。
- 真实需求：网页端要持续交互、流式彩色输出、Tab 补全、方向键、Ctrl+C、全屏程序（vim）——必须是真实 PTY。

## Requirements

### 功能需求
- **R1** 服务监听 `0.0.0.0:32677`，局域网可达（非仅 127.0.0.1）。
- **R2** 提供 WebSocket 终端端点，每条连接创建一个真实 PTY（node-pty），双向转发：终端输出→浏览器、键盘输入→终端。
- **R3** bridge 直接托管终端网页：浏览器访问 `http://<本机IP>:32677/` 即得到终端界面，无需拷贝文件、无需外网（离线可用）。
- **R4** 终端网页基于 xterm.js，支持：流式输出与 ANSI 颜色；键盘输入（含 Ctrl+C、Tab、方向键）；窗口尺寸自适应（fit addon）并把尺寸同步给 PTY（resize）。
- **R5** 默认 shell 为 Git Bash（复用 `getGitBashPath` 探测），探测不到时回退 PowerShell → cmd → sh；支持 `--shell` 参数覆盖。
- **R6** 保留「每次 CLI 调用自动后台拉起 bridge」机制，**不新增独立命令**。
- **R7** 在服务**首次拉起成功**时于主进程打印局域网访问地址（本机 IPv4 列表 + 端口）、网页地址与无鉴权风险提示（已运行不重复打印）。
- **R8** 支持同时多终端会话（多标签/多设备各自独立 PTY）。

### 非功能 / 约束
- **C1 安全**：MVP 不启用鉴权（用户选择，默认信任内网）。代码结构必须预留鉴权钩子（WS 连接校验），便于后续一行开启。
- **C2 安全提示**：服务首次拉起打印与网页首屏须明确提示「局域网内任何人可执行命令」的风险。
- **C3 打包**：node-pty 为原生模块，必须作为 `external` + `dependencies` 处理，确保全局安装后可运行。
- **C4 兼容**：保留健康检查端点（供 `startServer` 活性检测，路径调整为 `/health`）。
- **C5 无测试套件**：项目无测试框架，以手动/浏览器验证为准。

## Non-Goals (MVP)

- 鉴权 / 多用户隔离 / 权限控制（结构预留，默认关闭）。
- HTTPS / wss 加密传输。
- 会话录制、审计日志持久化。

## Acceptance Criteria

- [ ] **AC1** 本机执行任意 jie 命令（触发拉起）后，另一台局域网设备浏览器访问 `http://<本机IP>:32677/` 能看到终端界面。
- [ ] **AC2** 终端可输入命令并实时看到流式、带颜色的输出（如 `ls`、`git log --oneline --color`）。
- [ ] **AC3** 支持 Ctrl+C 中断正在运行的命令、Tab 路径补全、方向键调历史。
- [ ] **AC4** 浏览器窗口调整大小时，PTY 列行数同步 resize，输出不错位。
- [ ] **AC5** 默认进入 Git Bash（探测失败时 PowerShell），`--shell` 可切换。
- [ ] **AC6** 同时打开 2+ 个终端页（或多设备）各自独立、互不干扰。
- [ ] **AC7** 关闭浏览器/标签后对应 PTY 进程被释放（不留僵尸）。
- [ ] **AC8** `pnpm build` 产物 `dist/bridge.cjs` 能独立运行，node-pty 正确解析（全局安装场景）。
- [ ] **AC9** 服务首次拉起时主进程打印本机局域网 IPv4 列表 + 网页地址 + 风险提示；服务已运行时不重复打印。

## Decisions (Review Resolved)

- **不新增独立命令**：服务由现有「执行任意 jie 命令即自动后台拉起」机制启动；访问地址在**首次拉起**时于主进程打印（用户确认）。
- **xterm 资源**：bridge 静态 serve（离线）。
